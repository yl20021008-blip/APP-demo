# Changelog

## v1.3.1 Performance

本版本在 v1.3 基础上做性能优化，主要解决 Streamlit Cloud + Supabase + 大词库时的卡顿。

### 核心优化

1. **数据库连接池优化**
   - 从每次查询重新连接改成小连接池复用；
   - 使用 `pool_pre_ping` 和 `pool_recycle`，减少 Supabase 连接失效问题。

2. **数据库初始化优化**
   - `metadata.create_all()` 不再每次刷新页面重复执行；
   - 同一进程内只初始化一次；
   - 自动创建常用索引。

3. **学习状态同步优化**
   - 从 Python 全量扫描词库改为数据库端 `INSERT SELECT`；
   - 大词库下给用户补齐学习状态更快。

4. **词库分页显示**
   - “我的词库”不再一次加载几千词；
   - 支持每页 50 / 100 / 200 / 500 条；
   - 支持按章节、状态、搜索词分页查询。

5. **学习统计 SQL 聚合**
   - 不再一次读取全部 review_logs；
   - 由数据库直接聚合总次数、正确率、每日统计。

6. **词汇补全中心优化**
   - 默认每批 10 个；
   - 默认关闭自动翻译；
   - 最近记录只显示 50 条；
   - 补全候选词由 SQL 过滤，不再全表加载。

7. **词库导入优化**
   - 导入时预览只显示前 200 行；
   - 批量导入只在全部文件完成后同步一次学习状态；
   - 导入过程会显示进度。

### 部署提醒

Streamlit Cloud 的 Main file path 仍然填写：

```text
main_app/app.py
```

不要改回 `app.py`。
